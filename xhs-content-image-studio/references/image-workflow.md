# 配图工作流

当用户希望“配图 / 生成图片 / 直接把文案变成可发小红书的图文素材”时，按下面流程执行。

## 默认规格

- 小红书图文默认用 `3:4` 竖版
- 如果用户明确说“信息图 / 横版 / PPT / 课程图”，再切到 `16:9`
- 一张图只表达一个核心信息，不把解释段落塞进图里
- 先规划图清单，再定每张图逐字文案，最后生成提示词

## 阶段 1：配图规划

先输出图清单，让用户确认“要几张图、每张图讲什么”。

规划原则：

- 封面图：只放一个大标题，强调结果或痛点
- 内容图：每张图只讲一个判断、方法或步骤
- 对比内容：优先用双卡对比
- 清单内容：优先用 3 卡或 4 卡
- 流程内容：优先用路线图或步骤卡

给用户的输出格式：

- 图数量：N 张
- 图 1：用途 + 一句话核心信息 + 建议版式
- 图 2：用途 + 一句话核心信息 + 建议版式

## 阶段 2：Copy Spec

图清单确认后，再输出每张图“图上逐字写什么”。这是后续提示词和 API 请求的唯一真值。

压缩规则：

- 每个分区只保留结论或动作，不保留解释段落
- 封面图：标题尽量控制在 `6-14` 个中文字符
- 内容图：每块建议 `1-2` 行，每行尽量 `<= 12` 个汉字
- 如果字太多，优先删背景说明，保留结论

给用户的输出格式：

- 图编号：图 1 / 图 2
- 图类型：封面 / 清单 / 对比 / 流程 / 讲义
- 画幅：`3:4` 或 `16:9`
- 版式模板：封面大标题 / 双卡对比 / 三卡洞察 / 流程路线图 / 通用信息图
- 文字预算：全图允许多少字
- 区域与逐字文案：
  - 标题：
  - 区域 A：
  - 区域 B：
- 图标或隐喻：

## 阶段 3：Prompt Pack

Copy Spec 确认后，才能生成提示词。

提示词必须完成的事：

- 锁定画幅、版式、风格和留白
- 原样放入 Copy Spec 中的逐字文案
- 明确“除指定文字外不要生成额外文字”
- 约束中文必须清晰、不要密集小字、不要复杂背景

输出格式要求：

- 每张图一个独立代码块
- 代码块外只保留 1-2 句说明
- 若用户要批量出图，再额外生成 `out/apimart.requests.jsonl`

## 版式选择建议

- 封面：大标题 + 1 个主视觉
- 对比：双卡对比
- 清单：三卡洞察 / 四卡清单
- 流程：路线图 / 步骤条
- 总结：通用信息图
